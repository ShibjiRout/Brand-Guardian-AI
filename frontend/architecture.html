03 — Request Data Flow
01
User Submits Video
Option 1: YouTube URL posted to /audit. Option 2: video file + URL + title + description posted to /audit-upload.
frontend → FastAPI
02
Indexer Node — Video Ingestion
Option 1 uses yt-dlp to download. Option 2 uses the uploaded file directly. Video is uploaded to Azure Video Indexer via REST API.
LangGraph Node 1 → Azure Video Indexer
03
Azure Processes Video
Azure Video Indexer extracts transcript (speech-to-text) and OCR (on-screen text). Server polls every 30s until state = "Processed".
Azure Video Indexer → insights JSON
04
Auditor Node — RAG Retrieval
Transcript + OCR embedded via text-embedding-3-small. Top-3 matching compliance rules retrieved from Azure AI Search vector store.
OpenAI Embeddings → Azure AI Search
05
GPT-4o Compliance Audit
Retrieved rules + transcript + OCR + video metadata sent to GPT-4o. Returns structured JSON with violations, severity, status, and final report.
GPT-4o → JSON audit result
06
Response Delivered
FastAPI maps graph output to AuditResponse model and returns JSON. Frontend renders violations by severity and final PASS/FAIL verdict.
FastAPI → Browser
